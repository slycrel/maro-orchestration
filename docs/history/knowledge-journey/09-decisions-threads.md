## Discoveries & aha moments

### Harness is the answer to increasingly action-biased models (07-03)
Three same-night incidents of delegated forks shipping past authorized scope (one committed steps 7–10 unprompted and implied a merge that never happened; one designed+merged when asked only "is this worth doing?") flipped the framing from "write better prompts" to: prose scope is not a boundary — the harness must impose structural isolation, separate should-we from how-would-we, and scale verification with autonomy. Jeremy, verbatim in GOAL_BRAIN Invariants: "new LLM updates, leaning even more into action, the solution is always the orchestration harness."
Evidence: GOAL_BRAIN.md at 15dc1c4, Invariants ~line 110 + Decisions "2026-07-03 (harness guardrails for increasingly action-biased models)" ~line 811; e92421f (second incident recorded), 1cb22ab (guardrail principles).

### "Filesystem vs real memory" was the wrong axis — the ACCESS layer is the gap (07-04)
Two sub-agent inventories proved the dreaded memory chunk was misframed: append-only JSONL at single-digit MB with a working decay/promotion engine was healthy. What was missing: who can ask (workers got ZERO memory), how (TF-IDF + char caps, zero runtime embeddings), and traversal (2,124 knowledge edges written, zero readers). A storage migration would have rewritten ~30 readers/writers and bought none of the five gaps.
Evidence: docs/history/2026-07-04-memory-decision-brief.md §2–§4 (G1–G5); 1a2c887 ("the avoided chunk"); GOAL_BRAIN Decisions 2026-07-04.

### Both bake-off candidates passed the contract 24/24 — and both lost anyway (07-07)
The shim-ratio insight: passing a contract suite is not adoption evidence. In both live trials the port's real semantics lived in OUR adapter lines (~230 of 288 for Mem0, ~330 for Graphiti), framework reduced to a storage pass-through — plus live disqualifiers no paper screen caught (Mem0's embedded-qdrant single-client lock, fatal for a forking orchestrator; falkordblite leaking ~150 orphaned redis daemons). Companion insight: the memory market builds chatbot-persona SaaS; the four house requirements (hierarchical scope, invalidate-never-delete, no daemons, no LLM in the storage path) are jointly absent from every candidate. Jeremy: "ok, I'm convinced; steal sounds good when we take the strengths we're looking for from all 3 and put them together."
Evidence: docs/history/2026-07-07-memory-bakeoff.md (Round 2 table); 20ff596, 43e8cf7, ba1410b, eef81cd; GOAL_BRAIN Decisions 2026-07-07 ~line 1080.

### Worker memory measurably works — and the A/B exposed ALL git guards silently dead (07-08)
Pooled 16-run A/B (closure 8/8 vs 7/8, blocked workers 0 vs 1, median tokens-in −29%, exhaustions balanced 10v10) earned the first evidence-based default flip. - **Experiment-gated shipping at its most disciplined:** the memory port + 24-test contract built BEFORE any candidate and never changed (no goalpost drift); no memory piece landed without its consumer in the same chunk (consumer-first rule); worker-slice default flipped only on a pooled 16-run A/B with confounds explicitly adjudicated and contaminated cells excluded by token signature (memory-bakeoff "Pedigree"; GOAL_BRAIN 2026-07-07; worker-slice-ab pooled section).
- **Radical simplicity of the memory module:** ~300 lines stdlib sqlite3+FTS5 behind a 5-verb port, JSONL source of truth, adapters interchangeable on disk, memory_* imports stdlib-only so extraction stays a copy — 1.0ms/append vs hundreds of ms–seconds for the framework candidates (MILESTONES -1; src/memory_sqlite.py).
- **Pedigree-first documentation:** every era brief opens with status frontmatter and the full decision chain "so nobody re-litigates it from memory"; each carries its own reproducibility predicates (done-vs-achieved restates its exact filter predicates inline).
- **Honest self-audit as a first-class deliverable:** Purgatorio ran verify-before-fix on its own findings; done-vs-achieved excluded the poisoned era and hand-adjudicated artifacts on disk rather than trusting recorded verdicts — the project graded itself down before anyone else could.

## Lost good ideas

- **The reusable "run this prompt with this persona" pattern** — Jeremy explicitly asked for the bake-off to run as a generalized, reusable dispatch shape (GOAL_BRAIN 2026-07-07: the docs-brain-trust shape, generalized). Executed by hand (three parallel research agents), never productized as a named verb/skill; no later arc picked it up. *Worth reviving:* it is exactly the auto persona+skill packaging leap already named as the next big leap, with two proven worked examples to generalize from.
- **The blind persona-panel retrieval-quality tiebreaker round** — a designed third bake-off round where personas blind-judge retrieval quality, skipped because both candidates fell on structural grounds ("If the recommendation is contested, that round is the right tiebreaker to run"). Never re-invoked even as retrieval-quality questions (paraphrase hit@5 ~15%) accumulated. *Worth reviving* as the standing evaluation instrument for any retrieval change — cheap, already designed, and its evidence file (paraphrase lane) already exists.
- **The fastembed ONNX + sqlite-vec semantic lane (~150 lines)** gated on "BM25 measured insufficient", with the gate's evidence already identified (both adapters ~15% hit@5 on paraphrase). Nobody has run the gate check since. *Worth reviving, probably:* re-run the paraphrase lane against organic worker-slice traffic; if organic queries are paraphrase-shaped, this is the cheapest quality win on the table.
- **Stage-5 provenance pointers / language-form demotion** (memory-brief G4b, thread-arch #8 residual): compiled rules carry pointers to their source lessons/outcomes so a model upgrade can "re-fight from the evidence that produced it." Resolved as "compiled cache, portable via regeneration" (81c6914) but the demotion path was left as open BACKLOG — and the brief's own warning was that this is "cheap NOW ... and impossible later (provenance not recorded is gone)." *Worth reviving urgently by its own argument:* every Stage-4/5 promotion without a provenance field is evidence permanently lost, and model swaps (the exact re-fight trigger) are now routine on this multi-box setup.
- **Graph read-side wiring as "a read-side wiring project, not a greenfield project"** — G2's framing: 2,124 edges + wiki-link neighbor machinery already on disk, needing only 1-hop expansion in inject_knowledge_for_goal. **Superseded 2026-07-13** (GOAL_BRAIN Decisions): tracing the actual data before writing read-side code found the premise wrong — all 2,124 edges connect link-farm import nodes to each other; 0 touch the 252 system-authored orchestration nodes. Read-side wiring would have injected noise. Disposition: NOT built. The era's *method* survives inverted: the cheap first step is reading what's already written — which is exactly what killed the idea.
