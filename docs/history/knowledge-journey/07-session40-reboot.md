## Architecture as it was

At 6405cd8 the repo is **poe-orchestration** (the Maro rename came 06-25), a single Python program with 112 flat src modules — emphatically not an operating system: no cron, no enabled daemons, `deploy/systemd/` units are per-machine templates only ("Program, not operating system" invariant; Compiled truth notes the heartbeat service "is not enabled/running").

- **Entry:** `handle()` in a 2,131-line src/handle.py, two lanes. The quick NOW lane learned honesty this era: autonomous NOW runs self-verify and demote to `incomplete`, `now_lane.escalate_to_director` default ON, slim outcome records (bf86325, 0d249f1, ed9580e). The AGENDA lane runs through a 5,191-line src/agent_loop.py (plan → step exec → closure verification, where a contradicted "done" becomes `incomplete`, 5b648b3). Run metadata carries a brand-new second dimension beside status: `goal_achieved`/`goal_verdict_*` — done and successful are separate facts (66cc88e).
- **Memory:** eight flat-JSONL substrates behind one brand-new seam, `recall()` (src/recall.py: "One question, one function: 'what do I already know that's relevant right now?'"). Three slices — dispatch (pure file reads at every task dequeue, `_METADATA_SCAN_CAP=200`), loop (the eight substrates `_build_loop_context` used to inline), navigator (contract only). A dispatch guard (handle.py:1846-1855) refuses a goal after ≥3 all-non-done attempts in 60 minutes, autonomous requeue path only.
- **Underneath:** knowledge_web.py tiered lessons where decay is strictly a read-time derivation (`raw=True` on every RMW path — the M1 corruption fixes, 2465ece); an in-process marker-gated `maybe_consolidate()` "dream cycle"; knowledge_lens.py's hypothesis→standing-rule promotion pipeline live for the first time (promotion-at-reinforcement, cross-tier dedup, a8eb7f0) plus the entropy layer — contested-rule injection gates, `refight_rule()`, `last_verified` staleness (5dd5267, bfe937c). Captain's log: 47 event types, size-gated rotation on `log_event` (e3ecf2a; first live rotation archived 18,345 entries, 6MB→552KB, zero loss).
- **Navigator — exists but does not act.** src/navigator.py pins the decision schema: six moves (extend/execute/fork/collate/close/escalate) plus `idunno` as admission-not-move with a cheap→mid→power tier chain; close is invalid while any child is undispositioned. navigator_prompt.py renders and decides; navigator_shadow.py replays historical runs and, as of fef9a6e, shadows every live autonomous dispatch decide-only, sharing the guard's RecallResult — every NAVIGATOR_DECIDED row is navigator-vs-pipeline agreement data. Acting cutover code shipped 2 minutes before the era freeze (4bf71d0) as a **per-class** enable — `navigator escalate/close can act`, default OFF; the per-move `act_moves=["escalate"]` redesign arrived in the era tail (3f90ad9, 06-21). 6405cd8 itself is the MILESTONES note: "enable gated on organic data."
- **Steering:** a 407-line GOAL_BRAIN.md at repo root (2af04cd) — Jeremy's invariants verbatim ("paraphrase is how telephone flaws start"), compiled truth with a verification basis per claim, append-only dated decisions, a Threads section as fan-out defense, and a precedence rule: when it disagrees with any other doc, it wins. Each run-dir gets a scaled-down per-thread copy seeded at creation (src/thread_brain.py, 4e35d70).
- **Governance:** Poe subprocesses carry POE_WORKER_RUN=1 and a pre-push hook blocks worker pushes to main (d677e73) — direct response to this era's governance event.

## Discoveries & aha moments

### The write side was alive; the lifecycle was dead (06-10)
Weeks of "self-improvement isn't working" dissolved into dead code, not bad design. Stores had 1,272 outcomes and 60 lessons (38 medium + 22 long), but: three latent corruption bugs (tier-blind decay eroding LONG lessons on every load; every RMW write persisting decayed scores — compounding rot; default limit=50 truncating stores on rewrite), consolidation had never run, standing rules could structurally never accrete (observe_pattern fired once per lesson but rules need 2+ confirmations, and the promotion it depended on was racy), and the entire Phase 44-45 post-loop self-reflection block had been dead six weeks via a swallowed NameError. Fix philosophy became decree: decay is a read-time derivation, never persisted; and a pyflakes suite test locks out the swallowed-undefined-name bug class forever.
Evidence: 2465ece (M1), a8eb7f0 (M2), 90f940b (M3); docs/history/ROADMAP_ARCHIVE.md:1199-1224; GOAL_BRAIN at 6405cd8, Compiled truth ("the write side was always live... but the lifecycle was dead"); tests/test_static_undefined_names.py.

### GOAL_BRAIN.md: redistribute trust into a human-readable anchor (06-10)
The project stopped trusting its own docs ("Docs are best-guess" invariant) and created a single compiled-truth artifact that wins over everything else. The load-bearing idea, quoted in the file itself: "we're not escaping LLM trust, we're redistributing it, so the human-readable goal-brain becomes the actual non-LLM anchor." Format rules split human-steerable sections (verbatim quotes only — anti-telephone) from system-maintained ones (every claim needs a verification basis). Defined by example, not spec: "this file IS the format."
Evidence: 2af04cd (M4); GOAL_BRAIN at 6405cd8, header + Format rules; ROADMAP_ARCHIVE.md:1225 (precedence rule wired into CLAUDE.md).

### Goal identity does not survive the requeue boundary — memory needs a dispatch-time hook (06-10)
Pressure-testing the new artifact against 478 real run dirs found the machine eating itself: plan-step fragments recirculated as top-level goals (~40 error/stuck runs traced to a handful of parents), the heuristic decompose fallback chopped goals on [.;] precisely when the LLM was down (rc=1 era), and the same goal ran ~25 times in 35 minutes with lessons on disk nothing consulted. Reframe: recall must fire at dispatch, not just at loop start — hence recall()'s dispatch slice, the attempt-guard, and origin-ancestry linkage from enqueue into run metadata.
Evidence: 6183ad8 (pressure test + fallback fix), 6631a57 (recall() v0), 869d7b5 (origin ancestry); GOAL_BRAIN at 6405cd8, pressure-test findings 1-4; docs/RECALL_DESIGN.md at 6405cd8.

### Entropy steering: decay-by-invalidation, not decay-by-disuse (06-11)
Jeremy's entropy thread reframed memory decay as a response to world-change, not time: crystallized artifacts rot because "X's interface will change, browsers will have new standards," not because they were wrong. His pinned first pass, verbatim: "the first pass of a decay is to add the existing mechanism + failure to the prompt to re-fight the battle; at worst we have better context, at best it's a slight tweak and we fix forward." Companion insight: reinforcement and validity are different signals — the most-reinforced rule is the most dangerous one at world-shift time (hence `last_verified`, distinct from `last_applied`). Principle: decay trust, never data.
Evidence: 737bd83 (verbatim capture), d978423 (pinned as v0), 5dd5267 (contested gate + refight_rule), bfe937c (last_verified); GOAL_BRAIN at 6405cd8, Intent + Decisions 2026-06-11; ROADMAP_ARCHIVE.md:1164-1186.

### The navigator is right and the pipeline is dumb — proven by adjudicated divergence, including a governance event (06-11)
Shadow round 2 (seeded random N=20, seed 42) answered round 1's selection-bias caveat: 0/6 false escalates on well-formed goals, plus a side finding that flipped a core assumption — 11/20 randomly sampled goals were decompose-chop debris, including most pipeline-"done" ones, so done-status is not goal-health ground truth. Then the live shadow adjudicated for real: vague test goal "improve things" → navigator said escalate 0.95, pipeline executed into a 4.09M-token run that pushed unreviewed (good) code to origin/main authored as Jeremy. Every adjudicated divergence in the era went navigator-right. Consequences: mechanical worker push guard same day; and within days the per-class cutover design (4bf71d0, in-era) was superseded by per-MOVE cutover (3f90ad9, era tail) — escalate first, because it defers to a human and can't assert a wrong resolution.
Evidence: b977b7e (round 2), fef9a6e (live shadow), d677e73 (push guard), 4bf71d0 (per-class cutover, default OFF) + e7123dd (--agreement table), 3f90ad9 (per-move redesign, 06-21); ROADMAP_ARCHIVE.md:1194-1197; GOAL_BRAIN at 6405cd8, Decisions 06-11 evening; memory/project_session40_audit.md (6/6 escalate divergences navigator-right).

### Status integrity: "done" meant "the code path returned" — and everything upstream trusts status (06-11)
Impossible-goal probes (run a nonexistent binary) showed the honest model saying "cannot be fulfilled" while the run recorded done in 18s — which poisoned recall priors, permanently disarmed the dispatch guard (it requires all-non-done), and fed the navigator poisoned inputs. Jeremy's written call: "done != successful, done just means complete... if we're using done as 'no good output, but I did it' that's a problem." Fix: process status and goal verdict as independently recorded dimensions — absent key = unverified, never failed — plus a quick-lane self-verdict. Verified-done-not-reported-done finally got a mechanism.
Evidence: bf86325 (NOW self-verdict + escalation ON), 5b648b3 (closure verdict gates status), 66cc88e + 00a8294 (done≠successful as metadata dimension, Jeremy's call recorded); GOAL_BRAIN at 6405cd8, Decisions "2026-06-11 (night)".

### The rc=1 mystery was two readable bugs, and lesson extraction had been silently dead since Phase 59 (06-10/11)
Two humbling never-trust-the-wrapper finds. (1) The long-standing "claude subprocess failed (rc=1)" blocker decomposed: the CLI can print a complete success JSON and still exit non-zero, and the adapter trusted exit code over payload while truncated error dumps buried the CLI's actual message ("Not logged in · Please run /login") — historical rc=1 mysteries were readable errors nobody could see. (2) Live runs revealed extract_lessons_via_llm had returned [] on every real run since typed lessons shipped: safe_list's str default silently dropped every lesson dict, and all existing tests fed legacy strings so the dict path had zero coverage. Same lesson both times: only live, organic runs surface this class of rot.
Evidence: 9160cff (M5, payload-first + is_error load-bearing); e47d160 / d088ca7 side-branch twin (safe_list fix; commit message documents the zero-coverage gap); GOAL_BRAIN at 6405cd8, Compiled truth + Decisions 06-11 evening.

(4bf71d0, e7123dd, 6405cd8; ROADMAP_ARCHIVE navigator round-2 entry.)

- **Verbatim-quote invariants as anti-telephone:** human-steerable sections may not be paraphrased, which preserved Jeremy's actual reasoning (entropy, litmus test) in a form still quotable today. (97a9b21, 737bd83.)
- **Made-calls labeled as unmeasured:** guard thresholds, fork cap 8, confidence semantics all recorded as "a made call, not measured — revisit against <named event> data" — instrumentation wired before opinion.

## What we believed then

- **"Decay-by-invalidation v0... Shipped same day for the rule layer"** (GOAL_BRAIN Decisions 06-11). The read side shipped; the loop was structurally dead twice over — no contradiction writer ever wired (still true), and the refight caller lacked an adapter until 2026-07-09. "Shipped" described code that could never fire in production.
- **"Cutover per decision class"** (navigator schema decision, 06-11; shipped as such in 4bf71d0). One day of live adjudication broke the design: cutover had to be per-MOVE because moves differ in blast radius — escalate defers to a human and can't assert a wrong resolution; close asserts resolution without running. Recorded 06-12 in the session-40 audit; landed as 3f90ad9.
- **That M1–M3 had fixed the learning loop** ("any self-improvement-isn't-working observations from May runs are explained by these dead paths" — flagged "unverified but likely" even then). Dead segments kept surfacing for weeks: safe_list zero-lesson extraction (in-era, live runs only), refight unreachability (07-09); the Verify→Learn arc didn't close until 2026-07-14. The durable lesson — positive evidence over absence-of-error — arrived later.
- **That GOAL_BRAIN would stay "short enough to inject whole"** (its own goodness criterion). 407 → 3,565 lines by 2026-07-21; the whole-file-injection premise quietly died while the precedence rule survived.
- **"done" as a trustworthy signal** — believed by every upstream consumer (recall priors, dispatch guard, navigator inputs) until the impossible-goal probes; corrected in-era, but the era began inside this belief and most of its night-arc was escaping it.

## Lost good ideas

- **The contested-rule re-fight loop — entropy made mechanical.** Contradicted rules inject as "verify before relying," refight_rule() re-derives them against captain's-log evidence (keep/revise/retire→hypothesis), max 3/cycle, full audit events. Read side built and tested (knowledge_lens.py:511-704); the writer was never wired, so no rule ever becomes contested — silently stillborn. **Worth reviving, cheaply:** wire contradict_pattern into the existing closure-contradiction and lesson-contradiction checks (the natural collision detectors); everything downstream exists and has tests. Jeremy's "most-reinforced rule is most dangerous at world-shift" concern is still unaddressed — token_explosion has 29 confirmations and has never once been re-verified against a changed world.
- **GOAL_BRAIN whole-file injectability.** Append-only Decisions grew the file 8.8x with no compaction pass like the one MILESTONES/ROADMAP got (docs/history/ROADMAP_ARCHIVE.md, archived 06-24). **Worth reviving:** apply the ROADMAP_ARCHIVE pattern — hot compiled-truth + active decisions under a line budget, settled history to docs/history/. The precedence rule only means something if sessions actually read the whole anchor.
- **The --agreement adjudication table as a general authority-granting protocol:** new autonomous capability runs shadow-first, accumulates per-move agree/diverge rows against the incumbent, enable flip is a one-line reversible config decision made by Jeremy with the table in hand. Not lost for the navigator, but never generalized as a named pattern; later arcs (lesson_inject A/B) reinvented adjacent shapes. **Worth naming and reusing:** the project's cleanest answer to "when does an agent earn a new power," and it pairs naturally with done≠successful verdict data.
- **"This file IS the format" — example-as-spec.** Artifact definition carried by a living instance, guaranteeing spec can't drift from practice. Partially survives in thread_brain.py's scaled-down grammar; later design docs returned to spec-first. **Worth keeping in mind** for future artifact formats (persona packs, skill manifests): ship the exemplar, not the schema.
