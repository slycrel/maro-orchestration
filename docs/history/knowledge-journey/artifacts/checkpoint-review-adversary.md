### 1. BLOCKER — Phase 0.5's graduate-or-kill gate cannot produce a trustworthy verdict as specified
**Claim:** The seeded battery (4-6 tasks, with-skill vs control, "graduates only if it beats control") is statistically vacuous and structurally biased toward the with-skill arm.
**Evidence:** (a) n=4-6 binary observations per arm cannot distinguish signal from noise; "beats control" is undefined (1 extra catch out of 5?). (b) The plants are verbatim instantiations of the skill's own checklist items ("a half-closed-loop bait: store+reader+no-writer" vs the skill's "does this store have a live writer?") — the battery measures checklist-recall, not judgment transfer, the same instrument-validates-the-wrong-thing class as the static-probe bias in `/home/clawd/.claude/projects/-home-clawd-claude/memory/project_closure_probe_bias.md` (closure "lucked into being right" via checks that never touched the real question). (c) The control arm is contaminated: any without-skill session on this repo reads CLAUDE.md → GOAL_BRAIN.md → auto-memory, all of which already name half the through-lines (half-closed loops, done≠verified, verify-the-reviewer are in `docs/KNOWLEDGE_JOURNEY.md`, committed, and in the memory index) — diluting any measurable delta. (d) "Iterate once or kill" on noise means both graduation and killing will be coin flips laundered as decide-by-benchmark — the exact failure the rule exists to prevent.
**Correction:** Use the repo's own known-unfixed instances as the blinded ground truth instead of (or alongside) synthetic plants: before chunks 2-4 fix them, `decisions.jsonl` (readerless), `contradict_pattern` (dead writer, `src/knowledge_lens.py:373`), and the playbook horizon bug are REAL violations sitting in the tree. Score whether a with-skill session surfaces them unprompted during ordinary review tasks vs control. This dodges synthetic-task bias and — bonus — makes the before-chunk-1 sequencing load-bearing rather than arbitrary (the ground truth evaporates once chunks 2-4 land). Pre-register the pass bar; if the delta is ambiguous, ship the skill as a non-gated pre-read (cost ≈ zero) rather than pretending a benchmark decided.
**Note on sequencing (attacked, survives):** the delay cost is small (serial, 4-6 small tasks) and with the correction above, before-chunk-1 is the only order in which the real-instance corpus exists. The "prose doesn't survive" objection is answered by the design itself (graduation to tests) — see finding 8 for the enforcement gap.
