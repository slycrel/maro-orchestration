# The Knowledge Journey

*A historic timeline of how this project's **thinking** changed — the discoveries and aha moments, era by era, each judged against today's architecture. Commissioned by Jeremy 2026-07-20, before the swarm-review implementation arc: "I trust your judgement but sadly, not yet your context."*

Living doc: append an era when one closes. Detail files live in [docs/history/knowledge-journey/](history/knowledge-journey/).

## How this was built (and how far to trust it)

Twelve era-archaeologist agents excavated git + docs + memory archives at pinned commits; twelve adversarial verifiers claim-checked each record (refuted claims per era: 0–4, all corrected or dropped in the files); twelve writers produced the detail files; a completeness critic then hunted gaps across the whole set — its confirmed findings (a missed decree cluster, one stale-advice contradiction, two cross-file contradictions, four unnamed through-lines) were verified against GOAL_BRAIN/git and folded back in. Channel history (Telegram, Grok, Codex) was mined separately into [side-channels.md](history/knowledge-journey/side-channels.md).

**Coverage caveats:** the Telegram export ends 2026-04-15 and Grok rounds 4–5 are not on disk, so eras 06–11 have *no conversational-channel mining* — their record is repo-and-memory only. Era 12 is first-person, written by the session it describes.

**Naming key** (three layers renamed at different times, a recurring confusion): the GitHub repo was `openclaw-orchestration` until 2026-06-26; the Python package was `poe-orchestration` from 03-11 (cdea33b) to 06-25 (8a8d378); both became `maro-orchestration`. The June deep-rename decree split the concepts: Maro = framework, Conductor = role, Poe = optional persona.

## Through-lines

The recurring shapes — each visible in three or more eras before anyone named it:

1. **Done ≠ verified-done** — the project's longest thread. Checkbox pretend-builds (00) → anti-pattern #10 as prose (01) → "compiles is not works" (04) → verification theater: never-run checks counted as passed (05) → the done≠successful status split (07) → the unjudged tri-state and fails-toward-doubt (09) → verdict_trust() (10). Every era re-learned it at a deeper layer.
2. **Half-closed loops** — read side live, write side dead, or shipped-but-flag-OFF. Decision journal with times_applied=0 (03) → Phase 64 complete, OFF, forgotten (05) → the 19-item action queue never executed (06) → contradict_pattern with zero callers, refight stillborn (07) → the write-only graph (09; premise later proven wrong, 07-13) → and at the channel layer, conversational decrees backed by no machine state (side-channels). Named as a *class* only in era 12. The part of an idea that fits in one session ships; the rest stays conversation.
3. **Verify the reviewer** — reviewer findings hallucinate at ~30–78% across five independent measurements: the 03-28 automated audit (02), the 6/10 self-audit (03), 2/14 adversarial findings (04), fabricated fork reports (08), Purgatorio's self-application (09), Poe's factual misses (12). Verify-before-fix is not a preference; it is the project's most-replicated empirical result.
4. **Visible → Reliable → Replayable** — born in a Telegram aside Mar 28 (side-channels; inside era 02's window but absent from the repo record) → the dashboard exposed as "a prop in a play" → record-mode + the rung ladder (08) → the run-visibility arc and public viewer (11). The maturity doctrine that sequences the roadmap started as a conference anecdote.
5. **Perspective rotation & zoom** — the inversion whiteboard (04) → signal-source rotation designed then lost (05) → the orphaned persistence-and-zoom brief (06) → the camera-rotation decree, tied to inference-not-prompting (10's decree cluster) → personas-stay and evidence-path lenses (12). Time blindness is one axis of the same thing: the model has all times and no now.
6. **The attention economy / delivery loop** — decision-gated pings (00) → ConversationChannel.ask designed (05) → the escalation-channel decree: the go-between IS the surface (10's cluster) → the delivery-loop decree: the user must hear the outcome in plain words where they asked (11). Still unbuilt: the effort-estimate + consent message.
7. **Entropy & decay** — Jeremy's entropy thread → decay-by-invalidation and re-fight-on-collision (07) — whose contradiction-recording writer never shipped, making this also a half-closed-loop instance. The swarm-review arc's chunk 4 is this thread's third revival.
8. **Steal-don't-adopt & decide-by-benchmark** — provenance-tracked stealing (02) → A/B-earned defaults and shadow-first cutover (07, 09) → the bake-off where both candidates passed the contract and lost anyway (09). The control case for the whole history: *values* circled and converged; *features* circled and orphaned.

## What the history tells the swarm-review plan

Distilled from the era files' `edges_for_plan` (full versions in each file); this is the input to the plan-revision checkpoint:

- **The census must count more than readerless stores.** Phase 64 (05) was fully tested and flag-OFF-forgotten; the push guard died silently for two weeks (09); the impact scanner never fired on production data (10). The wiring-census tripwire needs three checks: every store with a reader has a live writer, every shipped flag has an enablement decision on record, every guard has a liveness test. The 05-12 md-claims census (06) is a ready spec — it ran once and died, which is itself the lesson.
- **Consumer-first rule for chunks 3–4** (10): no verdict/decision emitter lands without its acting consumer in the same chunk — V2 shipped with verdict_trust(), V5 shipped lesson+injection+flag together. The decisions.jsonl writer must land with its recall consumer proven live, or it becomes instance #7.
- **Chunk 4's writer already has a design precedent**: era 07's actual proposal is wiring contradict_pattern into the existing closure-contradiction and lesson-contradiction checks ("the natural collision detectors") — everything downstream (contested tier, refight) is built and tested. Review-findings-as-writer is a separate, newer proposal, not era-07-grounded. *(Corrected 2026-07-21: the first version of this line was itself a citation inversion, caught by the clean adversarial review.)*
- **Chunk 5 should rotate evidence channels, not judge personas** (05's signal-source rotation, independently reinvented by the current plan) and adopt the 05-12 error taxonomy as named lenses: phantom symbols, citation *inversion*, theory-mechanism confusion (06). Keep one context-blind reviewer (04's fresh-eyes institution); pin read-and-judge calls no_tools (11); cheap models never overrule a stronger model's UNDECIDED (11). The persona-triad ablation (04, never run) is the cheap sizing test before paying for N lenses.
- **Chunk 7 has prior art**: duty-cycle as operational success — era 06's point, paraphrased: a green status line can coexist with zero real work done (06 rated reviving it "partially"); the --agreement adjudication table as the authority-granting protocol (07); shadow-eval agreement batches as reviewer-drift sensor (11).
- **Standing doctrines that bind the whole arc**: reviewer reports are claims, never facts (08 forks; 09 harness-over-prose); inconclusive is a first-class verdict, not failure (09); archive-not-delete for all review artifacts (09); ship exemplar artifacts, not schema docs (07); after each confirmed finding class, a pin test makes it unrepresentable (07, 10).
- **Candidate revivals surfaced for discussion** (not in the current plan): decision-gated pings as the escalation shape (00); Also-After hooks as the structural attachment point for post-goal review (01); RISKS.md as reviewer input so swarms don't re-flag accepted risks (00); the blind persona-panel tiebreaker for contested verdicts (09, designed never productized); "run this prompt with this persona" as a first-class owned pattern — re-improvised by hand at least 4 times (09, 11); typed finding codes — the 05-12 review-error taxonomy (06) supplies the vocabulary; era 01's ERROR[E_*] codes are the greppable-runtime-codes precedent, not a review taxonomy; effort-language consent before launching a swarm (11).

## The meta-finding

Jeremy's 07-20 worry — "we keep circling ideas and only partially implementing things" — is confirmed by the record, and the mechanism is now legible: ideas enter as conversation, the session-sized fragment ships, and nothing structural carries the rest. The cure has been converging from both ends for months without being named: GOAL_BRAIN/SF-13 persist decrees as machine state (the channel-layer fix), and the wiring census + consumer-first rule close the code-layer half (the repo-layer fix). The history's single strongest lesson: **a principle only survives here when it is compiled into something a test can fail** — everything else waits to be rediscovered.
